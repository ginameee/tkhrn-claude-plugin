# Frontend Code Quality Review

프론트엔드 코드를 4축(가독성·예측 가능성·응집도·결합도) + 구조/에러로딩/데이터패칭/금지패턴 체크리스트로 리뷰한다. **코드를 직접 수정하지 않고** 개선점을 정리하여 보고한다.

## 리뷰 대상 결정

- 인수가 있으면: `$ARGUMENTS` (파일 경로, 디렉터리, PR 번호, 또는 범위)
- 인수가 없으면: `git diff`로 현재 변경사항을 리뷰
- PR 번호 형태(`#123`)가 들어오면 `gh pr view` + `gh pr diff`로 PR 내용을 가져와 리뷰

## 리뷰 절차

1. 대상 코드를 읽고 전체 구조를 파악한다
2. [antipatterns.md](antipatterns.md) 의 체크리스트 항목별로 위반 여부를 점검한다
3. 위반이 발견되면 해당 항목의 Bad/Good 예시를 참고해 구체적 개선안을 제시한다
4. 위반이 없는 항목은 생략한다

## 체크리스트 카테고리

1. **가독성** — 분기별 코드 분리, 구현 상세 추상화, 합쳐진 함수 쪼개기, 조건에 이름 붙이기, 매직 넘버, 시점 이동, 삼항 연산자, 비교 순서
2. **예측 가능성** — 이름 충돌, 반환 타입 통일, 숨은 부수효과
3. **응집도** — 디렉터리 구조, 매직 넘버 공유, 폼 응집도
4. **결합도** — 단일 책임 위반, 중복 허용, Props Drilling
5. **디렉터리 구조** — Atomic Design 레벨, 페이지 로컬(`_components/`, `_hooks/`), 공통 승격, Provider 위치, layout composition
6. **에러/로딩 패턴** — ErrorBoundary 계층, Suspense 페어링, `useSuspenseQuery`, AsyncBoundary, 스켈레톤 크기, 복구 액션, 낙관적 업데이트 롤백
7. **데이터 패칭** — debounce/throttle 위치, 순수 유틸 여부, TanStack Query `signal` 전달, abortRef 패턴, AbortError 처리, 비즈니스 훅 캡슐화, DTO→도메인 변환
8. **금지 패턴** — God Component(300줄+), `any` 타입, 페이지 컴포넌트 비즈니스 로직, 비동기 에러 `useState`, 매 렌더 새 Promise, atoms/molecules 외부 의존

## 출력 형식

```markdown
## 코드 품질 리뷰 결과

### 요약
[전체 코드 품질 1-2줄 요약]

### 발견된 이슈

#### [카테고리] — [항목명]
- **위치**: `파일:라인`
- **카테고리**: 가독성 / 예측 가능성 / 응집도 / 결합도 / 구조 / 에러로딩 / 데이터패칭 / 금지패턴
- **문제**: 구체적으로 무엇이 왜 문제인지
- **개선안**: Before/After 코드 포함
- **우선순위**: 높음 / 중간 / 낮음

(반복)

### 잘된 점
[좋은 패턴이나 구조적으로 잘한 부분]

### 트레이드오프
[선택의 근거가 있는 경우 명시. 모든 지적이 절대 규칙은 아님]

### 총평
- 총 N개 이슈 발견
- 가장 우선 개선할 항목: ...
```

## 리뷰 원칙

- **심각도 순서**: 예측 가능성·결합도 > 데이터패칭·에러로딩 > 가독성·응집도·구조 > 금지 패턴(즉시 수정)
- 위반이 없으면 "이슈 없음"으로 짧게 끝낸다
- 주관적 스타일 취향은 지적하지 않는다. 체크리스트 항목에 해당하는 것만 지적한다
- 개선안은 반드시 구체적 코드를 포함한다
- 지적 건수 부풀리기 금지 — 중요한 문제에 집중한다
- 프레임워크 의존 코드(Next.js App Router 등)는 해당 프레임워크 기준으로 판단한다
